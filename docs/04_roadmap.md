# 版本路线图：V0 - V5

每个版本 = 一个 Git tag = 一个可运行的完整版本 = 一次迭代。

## V0：开山之版 —— 通用脚手架跑通

**目标**：把 LÖVE2D 的基础链路跑通，建立可复用的 core 层骨架。
**标签**：`v0.0.1`
**预计工作量**：约 1-2 天
**为什么先做这些**：地基不牢，地动山摇。先把通用能力做出来，后面做象棋业务时就不用边做边改基础设施。

### 范围
- ✅ LÖVE2D 启动链路
- ✅ 场景切换（至少两个场景验证）
- ✅ 输入层（键盘 + 鼠标，动作映射）
- ✅ 资源管理（图片缓存加载）
- ✅ 渲染分层（5 层结构）
- ✅ 日志系统（分级 + 格式化）
- ✅ 配置系统（默认值 + 读写）
- ✅ 事件总线（发布/订阅）

### 代码文件
```
conf.lua
main.lua
src/core/init.lua
src/core/logger.lua
src/core/config.lua
src/core/event_bus.lua
src/core/input.lua
src/core/scene_manager.lua
src/core/resource_manager.lua
src/core/render_layer.lua
src/core/utils.lua
src/game/init.lua
src/game/scenes/boot_scene.lua
src/game/scenes/test_scene.lua
```

### 验收标准
1. 执行 `love .` 能正常启动，显示启动画面
2. 按某个键（如空格）能切换到测试场景
3. 再按一次能切回来，证明场景切换工作
4. 移动鼠标，屏幕上显示鼠标坐标（证明输入系统工作）
5. 渲染 3 个不同层的图形，确认层级正确
6. 控制台有日志输出，不同级别有不同颜色/格式
7. 加载同一张图片两次，第二次走缓存（日志证明）
8. 退出游戏不报错误

## V1：视觉框架 —— 棋盘 + 主菜单

**目标**：把中国象棋的视觉骨架搭起来，能看到棋盘和菜单。
**标签**：`v1.0.0` ✅ 已完成
**状态**：已实现并通过本地测试
**为什么先做视觉**：有画面才有反馈，才有继续开发的动力。而且视觉是需求最明确的部分。

### 范围
- ✅ 棋盘渲染（10 行 × 9 列，楚河汉界）
- ✅ 主菜单场景（开始游戏、设置、关于）
- ✅ 按钮 UI 组件
- ✅ 字体系统
- ✅ 场景过渡动画（淡入淡出）
- ⚠️ 棋子占位（先放方块代替，V2 再做真棋子）

### 代码文件（新增）
```
src/game/scenes/menu_scene.lua
src/game/scenes/game_scene.lua
src/game/entities/board.lua
src/game/ui/button.lua
assets/fonts/ (至少 1 个中文字体)
```

### 验收标准
1. 启动后进入主菜单，有 3 个按钮
2. 鼠标悬停按钮有反馈（变色或放大）
3. 点击"开始游戏"进入对局场景
4. 对局场景中显示完整棋盘（线条 + 楚河汉界）
5. 按 ESC 能返回主菜单
6. 场景切换时有淡入淡出效果
7. 棋盘居中显示，不同分辨率下适配正常

## V2：核心玩法 —— 棋子 + 走法 + 吃子

**目标**：能正常下一盘棋，规则基本完整。
**标签**：`v2.0.0` ✅ 已完成
**状态**：已实现并通过单元测试（32 个用例，全部规则正确）
**为什么这一版做核心玩法**：游戏的本质是交互，V1 有了视觉骨架，V2 要让它"能动"。核心玩法是最重要的，必须尽早验证。

### 范围
- ✅ 棋子实体（7 种棋子，红黑各一套）
- ✅ 棋子初始摆放
- ✅ 选中棋子（点击高亮）
- ✅ 可走位置提示
- ✅ 走棋（点击目标位置移动）
- ✅ 吃子（吃掉对方棋子，从棋盘移除）
- ✅ 走法规则校验（车、马、象、士、将、炮、兵）
- ✅ 回合切换（红方走了换黑方）

### 代码文件（新增）
```
src/game/entities/piece.lua
src/game/systems/rules.lua
src/game/systems/game_state.lua
assets/images/pieces/ (棋子图片)
```

### 验收标准
1. 棋盘上有完整的 32 颗棋子，初始位置正确
2. 点击红方棋子，棋子高亮，显示可走位置
3. 点击可走位置，棋子移动过去
4. 移动后轮到黑方
5. 所有 7 种棋子的走法规则正确
   - 车：横竖走任意格，不能越子
   - 马：走日，有蹩马腿
   - 象：走田，不能过河，有塞象眼
   - 士：走斜线一格，不能出九宫
   - 将：走一格，不能出九宫，将帅不能对面
   - 炮：走法同车，吃子需隔一个子
   - 兵/卒：前进一格，过河后可横走，不能后退
6. 吃子后被吃的棋子消失
7. 非法走法被拒绝（棋子不动，有视觉反馈）

## V3：胜负判定 + UI 完善 + 动画

**目标**：一局棋有始有终，界面完整可用，有动画反馈。
**标签**：`v3.0.0` ✅ 已完成
**状态**：已实现并通过测试
**为什么这一版做这些**：V2 能下棋但没有"结束"，也没有完整的 UI。V3 要让游戏体验完整闭环，加上动画让游戏感更足。

### 范围
- ✅ 将军检测
- ✅ 将军 / 被将军提示
- ✅ 将死判定（胜负判定）
- ✅ 游戏结束界面
- ✅ 悔棋功能（可选，看复杂度）
- ✅ 设置界面（音量、分辨率）
- ✅ 棋盘坐标显示
- ✅ 当前回合提示
- ✅ 走棋动画（平滑移动）
- ✅ Tween 动画系统（core 层新增）

### 代码文件（新增）
```
src/core/tween.lua
src/game/scenes/settings_scene.lua
src/game/ui/panel.lua
```

### 验收标准
1. 将军时有明显提示（文字 + 特效）
2. 将死时弹出胜负界面
3. 胜负界面有"再来一局"和"返回菜单"按钮
4. 设置界面能改窗口大小，且生效
5. 棋子移动时有平滑动画（不是瞬移）
6. 棋盘边有坐标（1-10, a-i）
7. 屏幕上方显示"红方回合/黑方回合"

## V4：AI 对手 + 音效 + 设置

**目标**：可以人机对战，有声音反馈，能调整设置。
**标签**：`v4.0.0` ✅ 已完成
**状态**：已实现并通过测试
**为什么这一版做这些**：有了完整规则后，单人对战需求最迫切。AI 让游戏可以自己玩，音效提升游戏感，设置让系统更完整。

### 范围
- ✅ AI 对手（初级难度，随机走合法步）
- ✅ AI 中级难度（简单搜索 + 估值）
- ✅ 难度选择界面
- ✅ 背景音乐
- ✅ 音效（走棋、吃子、将军、胜负）
- ✅ 粒子系统（core 层新增，选子特效、吃子特效）
- ✅ 选中效果动画（呼吸、发光）

### 代码文件（新增）
```
src/core/particle.lua
src/game/systems/ai.lua
assets/sounds/ (音效文件)
assets/music/ (背景音乐)
```

### 验收标准
1. 主菜单可以选择"人机对战"或"双人对战"
2. 人机对战时，电脑会自动走棋
3. 初级 AI 不会犯规，但比较菜
4. 中级 AI 有基本的策略（吃子、保子）
5. 走棋有"嗒"的音效
6. 吃子有"啪"的音效
7. 将军有特殊提示音
8. 背景有轻音乐
9. 吃子时粒子特效爆开

- ✅ 构建脚本 `build.sh`（一键打包 .love + Linux 可执行）
- ✅ Steam 发布流程文档

## 版本规划总览

```
V0 ── 通用脚手架 ──┐
V1 ── 视觉框架 ─────┤
V2 ── 核心玩法 ─────┼──→ 游戏可玩
V3 ── 胜负 + UI ────┤
V4 ── AI + 音效 ────┤
V5 ── 发布准备 ─────┴──→ 可以发售
```

## 版本号规则

- `v0.x.x`：开发中版本，API 可能变
- `v1.0.0`：第一个有实际游戏内容的版本
- `vX.Y.Z`：X = 大版本，Y = 小版本，Z = 补丁
- 每个正式版本打 tag，格式 `vX.Y.Z`
- tag 对应可运行的完整版本
