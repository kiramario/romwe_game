# 项目愿景：Trillion Games 2D Engine

## 一句话定位

一个由独立开发者掌控的、可复用的 2D 游戏引擎脚手架，首发产品为中国象棋，远期可扩展到弹珠、卡牌、解谜等多种 2D 游戏类型。

## 核心理念

1. **代码完全可控**：不依赖黑盒引擎编辑器，所有逻辑在代码中。开发者理解每一行代码的作用。
2. **架构分层清晰**：通用引擎层（core）与具体游戏层（game）严格分离。换游戏 = 换 game 目录，core 不动。
3. **迭代式开发**：每个版本都有可玩、可验证的成果。不做"半年憋大招"式开发。
4. **视觉有质感**：虽然是 2D，但通过分层、光影、粒子、镜头运动营造纵深感和品质感。不做真 3D。
5. **发布优先**：从 V0 就考虑多平台发布路径。目标平台优先级：Steam > 桌面 > Android APK。

## 首发产品：中国象棋

为什么选中国象棋作为第一个游戏：
- 规则明确，逻辑可验证，适合作为引擎的"测试用例游戏"
- 2D 俯视视角，天然适合验证渲染分层、输入系统、状态管理
- 受众明确，Steam 上有同类产品可参考交互设计
- AI 难度可调，延长游戏寿命
- 开发周期可控，不会陷入内容生产的无底洞

## 远期愿景

- core 层演进为一个轻量但完整的 2D 游戏框架
- 通过替换 game 层快速产出第二款、第三款游戏
- 形成"Trillion Games"品牌下的小游戏系列
- 所有游戏共享同一套发布流水线、存档系统、成就系统

## 非目标（不做什么）

- ❌ 不做 3D 游戏
- ❌ 不做大型多人在线游戏
- ❌ 不做通用商业引擎（只服务于内部项目）
- ❌ 不追求极致性能优化（先跑通，再优化）
- ❌ 不在 V1 之前考虑编辑器化
